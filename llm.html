LLM, from First Principles

半ば形だけの質問をチャット欄に貼り付けると、一瞬で丁寧な同僚のような流暢な段落が返ってくる。引用を頼むと、存在しない論文名がもっともらしく並ぶ——著者名、年号、ジャーナル風の語彙まで整っているのに、文献データベースには何もない。捏造引用は、悪意ではなく、最初に触れるべきミステリーの一つだ。単語の文字数を数えさせると、自信ありげな口調のままつまずく。同じシステムが法律っぽい文面を書き、量子力学を比喩で説明し、それから静かに「なかった会議」の議事録を捏造する。摩擦は「機械がバカだ」ということではない。流暢さと真偽追跡が、日常会話が教えてくれなかった形で切り離されたということだ。

もう一つ、再生成ボタン。同じ質問でも、二回目は言い回しが変わる——比喩が入れ替わり、箇条書きの順序が入れ替わる。それでも結論の骨格はしばしば同じ。再生成の言い回し差は、モデルが「考えを改めた」のではなく、別の確率的経路を辿ったサインだ（第6章で回収）。三つ目：長い契約書を丸ごと貼り、冒頭に「必ず三項目だけ答えよ」と書いても、応答が五項目になったり、禁止した形式で返ってきたりする。長い貼付で先頭指示が消える——これも第8章の有限コンテキスト窓で回収する。

第一原理の地図がなければ、印象的な回答は魔法か詐欺のどちらかに見え、どこを信じ、何を検証し、失敗時にどの層が壊れたのかを決める力を失う。ここでのリテラシーは能動性だ——幽霊と交渉するのをやめ、制約された予測機械として扱い始める。プロンプトを選び、主張を確かめ、ボトルネックがトークン化なのか、コンテキストなのか、サンプリングなのか、訓練目的そのものなのかを見分ける。

この記事は「LLM の正体は Transformer だ」というスローガンから始めない。スローガンは出口に近い圧縮に過ぎない。入口は、あなたがすでに体験している違和感——捏造引用、再生成の言い回し差、長い貼付で先頭指示が消える——から始める。各章末尾の ### 出口命題 は、その層が保証することと保証しないことを一行で固定するチェックポイントだ。機構名は、それらの違和感を説明するために、問題が迫った順で登場する。

この記事の狙いは、制約から機構を再構成することだ。必要になるまで専門用語を遅らせ、各層を迫った問題の物語順にたどる。読み終わったとき、あなたは「AI が何を“考えている”か」ではなく、「今どの層で、何が入力され、何が最適化され、何が検証されていないか」を声に出せる——それがポケットマップの中身だ。

このツアーでやること / やらないこと

離散テキストから嗜好で形作られたチャットまで、発明順の制約連鎖を再構成する。各章は新パーツを少数だけ導入し、分布を体感しないと理解できない箇所だけ対話型の玩具を置き、失敗を層で特定する判断規則で終える。プロンプト料理本は出さない。AGI 予言もしない。嗜好調整＝真実性とも言わない。

対話型の玩具（tokenize-embed、next-token、attention-mix、temperature-sample、context-window）は、断言を暗記させるためではない。スコアベクトル・ソフトマックス重み・窓の切り詰めを触って、段落の比喩が指している対象を指先で確認するためだ。図（token-pipeline、attention-schematic、autoregressive-loop、layer-map）は、症状から層へ降りるときのポケット地図の折り目になる。

読み終えたあと、あなたがやめるべきこともはっきりさせる。モデルに「今度こそ嘘をつくな」と感情で懇願すること——それは嗜好つき方針や分布的もっともらしさの層をすり抜け、幽霊交渉に戻る。逆に、すべての出力を無視することも、層地図の放棄だ。ブレストと検証可能な主張の間には、外側の検証ループを差し込む余地がある。

理解の道筋（問題の列）

  1. 打ち込まれた言語は、いかにして計算可能なものになるのか？
  2. その表現を有用な予測へ変える訓練圧は何か？
  3. 有限コンテキスト窓の中で、後の語はどうやって先頭の指示の引きを感じるのか？
  4. スコアの雲は、いかに一つの発話列になるのか——再生成で言い回しが変わるのはなぜか？
  5. 文書続きエンジンは、なぜ汎用タスク界面、さらにチャット助手のように見え始めるのか？
  6. 流暢さが嘘をつくとき、最初に疑うべき層はどれか？

（問題は六つ。機構章は九つ——入口の契約、問題列、六つの機構、締めのポケットマップ——だが、問いの数は六に抑えている。）

この六問は、部品名のチェックリストではない。読者が実際に困る順——文字列をどう数にするか、何を学習しているか、遠い指示をどう届けるか、返答をどう一列にするか、助手らしさをどう得るか、失敗をどう診るか——に並べた理解の道筋だ。各章は一問に答え、新しい部品を三つ以内に抑え、出口命題で次の問いへ橋を架ける。

入口で三つの日常ミステリーをポケットに入れておく。捏造引用——完璧に聞こえるのに文献がない。再生成の言い回し差——要点は同じなのに形容が入れ替わる。長い貼付で先頭指示が消える——冒頭のルールが後半で蒸発する。それぞれ、該当する層に到達したときに回収する。

各問題は、次の章タイトルが機構名ではなく問題の名前になっている理由でもある。読者が持ち帰るべきは部品表の暗記ではなく、症状から層へ降りる道筋——理解の道筋——だ。

まず言語を数にしなければならない

チャット欄に文を打つと、返答は言語に見える。だが下層では、人間のように文字や意味を「保持」しているわけではない。ニューラルネットは行列積の積み重ねであり、入力はすべて固定長の実数ベクトル、演算は予測可能な形状のテンソルを要求する。生のチャットテキストは離散記号——Unicode 文字、空白、句読点——であり、重み行列にそのまま入れられない。最初の設計上の失敗は率直だ。機械は「単語」を掛け算できない。 パイプラインの次段は、文字列を有限リストで索引づけられた数に変換しなければならない。

ここでよくある誤解を先に潰しておく。モデルが「単語の意味を理解している」から賢いのではない。理解という語は、後段の層——次トークン予測、自己注意ミックス、嗜好つき方針——が積み上げた振る舞いに、読者が後付けでラベルを貼っているに過ぎない。第3章の仕事は、その振る舞いの最下層を固定することだ：入力は必ず整数列になり、各整数は埋め込み参照でベクトルに変換される。それ以外の経路はない。

トークン語彙

最初の手は、キーボードストリームを計算単位とみなすのをやめることだ。モデルはトークン語彙——トークナイザが出力しうる文字列片の閉じたリスト——を定義する。トークンはよく出る語全体（" the"）か、語尾片（"ing"）か、稀な綴りでは一文字かもしれない。語彙サイズは訓練時に固定され、しばしば数万エントリ。実行時に語彙外記号は出ない——未知テキストは既知の片に切られる。トークンは再利用可能な文字列片であり、概念ではない。 "bank" が一トークンでも、川岸か金融かの意味は載らない。訓練テキストにしばしば現れる、索引可能な部分文字列に過ぎない。

同じ英文でも、トークナイザの世代やベンダーで分割が変わる。"ChatGPT" が一トークンか "Chat" "GPT" かは実装依存だ。コードや URL、人名の綴りでは、一文字ずつに割かれることもある。だから「この単語をモデルは一塊で見ている」と決め打ちできない——トークンID 列を確認しないと、プロンプトの微妙な空白や句読点が能力評価を歪める。

トークンID

語彙各エントリにはトークンID——0 から V−1 までの整数——が割り当てられる。"Hello" が語全体なら一つの ID、分割されれば複数 ID。モデルが見るのは文字ではなく ID 列だけだ。チャットの一ターンは、順序付き整数配列として保存される。パディングやバッチ化で行列に包まれるが、意味論的なコミットはここで済んでいる——言語は、いかなる「理解」が仮説化されるより前に、ID の有限アルファベットへ離散化された。

埋め込み参照

行列機械は依然としてベクトルを要する。裸の整数ではない。埋め込み参照表は形状 [V, d] の行列 E だ。行 i は ID i に対応する d 次元ベクトル。一 ID の順伝播は索引：vector = E[id]。ID 列に対しては、下流が混ぜられるベクトル列が返る。決定的なのは、これが百科事典からの検索ではない点だ。埋め込み参照は参照表であり、真理の図書館ではない。 行はランダム始まりで、後続の訓練圧だけが形作る。この層では、似た ID がベクトル空間で近接する保証はない。

訓練が進むと、似た文脈で交換可能なトークンは近づきうる——だがそれは後段の次トークン予測圧の副産物であり、埋め込み参照が「意味辞典」を実装しているわけではない。行 i を取り出す操作は、データベースの JOIN でもナレッジグラフのルックアップでもない。索引 i → ベクトルの一点参照だ。

再構成：文字列からテンソルへ

プロンプト断片 "The cat sat" を考える。トークナイザが三 ID に割り当てたとする。

トークン片トークンID
"The"464
" cat"3797
" sat"3331

埋め込み表（例として d = 4）は次を返す。

E[464]  → [ 0.12, -0.05,  0.33,  0.08]
E[3797] → [-0.21,  0.44,  0.02, -0.19]
E[3331] → [ 0.07,  0.11, -0.28,  0.41]

積み上げると 3 × 4 行列——三ベクトルが列順——がモデルに入る。Unicode も、構文木も、命題内容もない。索引付き行が演算の準備完了だ。系列モデル向けのニューラル機械翻訳が、この ID→埋め込みのパイプラインを確立した（Bahdanau et al., 2014）。GPT-2 のような大規模言語モデルも、入り口のインターフェースは同じで、語彙と文脈だけが広い（Radford et al., 2019）。

バッチ処理のイメージ：複数ユーザーのプロンプトは、長さを揃えるため末尾に パディング ID が付く。埋め込み参照はパディング行も持つが、注意マスクで無視される。ここでも「意味のないパディング」が混ざる——モデルが見るのは常に ID 列という契約は変わらない。プロンプト注入攻撃や不可視 Unicode の話は、セキュリティ層の話だが、根底は同じ：文字列→トークンID の境界が、意図と実入力をズラしうる。

シミュレーションを開く — 言語は思考として入るのではなく、語彙片→整数ID→埋め込み参照の列として入る。

出口命題

言語は、思考としてではなく、トークンID 列がベクトルへ写像された列としてモデルに入る。

ベクトルは存在するが、この段階では " sat" の次にどの ID が来るべきかは何も言っていない。E の行は、訓練圧が実テキストの統計構造に結びつけない限り任意に漂いうる。次の失敗は即座に来る——ベクトルには「次に来る ID」を言う圧力がまだない。

圧力は「次のトークンは何か？」

第3章は、トークンID で索引された浮動小数点行で終わった——微分可能でバッチ可能だが、意味論的には無拘束。埋め込み表をランダムに置き換えても、訓練目的がなければ架構は文句を言わない。失敗は「モデルに世界知識がない」ではない。もっと狭い——局所目的がなければ、埋め込みは汎用语言利用のために自己配置する理由を持たない。 現代の言語モデルスタック全体は、この隙間への一つの答えの上に建っている——タスクヘッドの群れでも、真偽基準でもなく、各位置で繰り返される一つの条件付き当てゲーム。

文脈を条件づけとして

トークンID 列の一位置に注目する。通常の左から右の設定では、左側すべてが文脈——次をスコアリングするときに条件とみなしてよい接頭辞。"The cat sat on the" は文脈；続くスロットでの仕事は、語彙全体に確率質量を配分することだ。損失は接頭辞が質問か命令か引用かを問わない。唯一のコミットは、早い ID が現在位置のスコアベクトルに影響すること。文脈を条件づけとして捉えると、一つのスタックが翻訳・要約・チャット風続きをモジュール交換なしに担える——接頭辞の形式がどの ID をありそうにするかを変えるだけで、機構は常に「前に来たものを与えて、次 ID をスコアせよ」。

たとえば接頭辞が User: ... Assistant: 型なら、続きは助手方言になりやすい。def で始まれば Python 続きになりやすい。Once upon a time なら物語調。これらは別タスク ID をネットワーク内で切り替えているのではなく、同じ次トークン予測が異なる条件付き分布を出しているだけだ。Radford らは、大規模な事前訓練だけで多様な続きスタイルが単一目的から立ち上がることを GPT-2 で示した（Radford et al., 2019）。

次トークン分布

位置 t における各順伝播の出力は次トークン分布——長さ V のベクトルで、非負かつ和が一。インデックス j は、接頭辞が t で終わる直後に語彙 ID j が来る確率として読む。実装ではネットワークは生の logits（未正規化スコア）を出し、ソフトマックスが確率にする。"The cat sat on the" では " mat" " floor" " couch" などの ID に質量が集中しうる——モデルが家具を「知っている」のではなく、訓練テキストで似た接頭辞の後にそれらの続きがしばしば現れたからだ。局所尤度はグローバルな真偽スコアではない。 高確率は「この ID がコーパスで似た文脈の後にしばしば続いた」という意味で、「世界においてこの記述が正しい」ではない。

生テキストへの交差エントロピー

訓練は、何十億もの実続きを見せて欠けていた圧を供給する。各位置で、文書からの正しい次 ID をターゲットクラスとし、生テキストへの交差エントロピーが誤った ID への確率を罰し、実際の次トークンへの確率を報酬する。位置と文書で平均すると、勾配降下が埋め込みと上流の重みすべてを、訓練でありそうな続きに尤度を与える方向へ押す。この段階に「有用」「事実」とラベルする人間はいない——教師信号は文字通り、同じ Web ページ・書籍・スレッドから切り出した次の文字かトークン。Bengio らは文脈からの語条件分布学習を形式化した（Bengio et al., 2003）。当時のニューラル言語モデルは、固定長の前語コンテキストから次語の分布を出す——現代 LLM の次トークン予測の直接の祖先だ。GPT-2 は同じレシピをスケールさせる——一つの Transformer、一つのソフトマックス頭、クロールテキスト上の一目的（Radford et al., 2019）。一目的から広い能力が立ち上がりうる——損失が各技能を明示符号化しているからではなく、多くの技能が「続きを予測せよ」に圧縮可能だから。

歴史的メモを層の言葉で結ぶ：Bahdanau の注意（2014）はエンコーダ–デコーダ間のソフトルーティング；Vaswani の自己注意（2017）はデコーダ内の全位置混合；GPT-2/3 はそのデコーダスタックに次トークン損失だけを掛けた。チャットの「助手感」は、Ouyang ら（2022）以降の指示＋嗜好層——いずれも同じトークンID 列の上に載った別圧として読める。

たとえば要約らしい続き、コードらしい続き、Q&A らしい続きは、すべて「この接頭辞のあと、訓練コーパスではこう続いた」という統計の産物として説明できる。モデルが「要約タスクを実行している」という内部フラグを持つのではない。タスク境界は、接頭辞のトークン列が与える条件付けにすぎない——第7章の文脈内パターン完了へ続く橋だ。

再構成：損失の一ステップ

玩具語彙 V = 5、ID {the=0, cat=1, sat=2, on=3, mat=4}。接頭辞 ID [0, 1, 2, 3]（"the cat sat on"）処理後、次スロットの logits が次のとおりだとする。

IDトークンlogitソフトマックス p
0the−1.00.02
1cat−0.50.03
2sat−2.00.01
3on−1.50.02
4mat+3.00.92

訓練文書は ID 4（mat）で続く。この位置の交差エントロピー損失は -log(0.92) ≈ 0.08——小さい。モデルが既にコーパスと一致しているから。真の次 ID が 1（cat）なら、同じ logits で -log(0.03) ≈ 3.5 という大きな罰が質量の配分を変える方向へ重みを押す。すべての位置・すべての系列で繰り返す。最適化されている技能は次トークン予測——この層ではそれ以上のエキゾチックなものはない。

大規模事前訓練のスケール感を数字で固定する必要はないが、イメージは持て：数十億トークン × 各トークン位置で一回ずつ損失——同じ「次を当てよ」ゲームの反復。GPT-2（Radford et al., 2019）は WebText 上でこれを走らせ、単一 Transformer で多様な続きスタイルを獲得した。GPT-3（Brown et al., 2020）はさらにパラメータとデータを増やし、文脈内パターン完了が実用域に入る閾値を示した——だが根底の損失は変わらない。

もう一つの反混同：モデルが「自信を持って」答えるのは、真偽に自信があるからではない。ソフトマックスの尖った分布は、この接頭辞のあとコーパス上よく来た続きへの尖りだ。引用符付きの欄外音「私は確信しています」も、訓練テキストで権威ある文体のあとに続きやすいトークン列なら、尤度は上がる。だから第8章の分布的もっともらしさへ直結する——流暢な自信と世界の真偽は、損失関数上で結びついていない。

シミュレーションを開く — 学習が押しつける技能は、左文脈からの次トークン予測であり、世界の真偽スコアではない。

出口命題

最適化されている技能は、生テキスト上の左文脈からの次トークン予測である。

その目的は局所的——各位置は左側すべてを与えられた直後の次 ID だけをスコアする。だがユーザーのプロンプトは、最初の文に二十トークン後の名詞を支配しなければならない制約を置きうる——主述一致、定義した略語、フランス語で答えよという指示。各位置の手作り特徴の固定袋ではその遠隔制約を信頼してルーティングできず、接頭辞全体を一つの再帰状態に圧縮する方法は系列が長くなるほど脆い。遠隔制約は有限窓内の各位置に届かなければならない——次章が扱う失敗はまさにそれだ。

RNN や LSTM は理論上、任意長の履歴を状態に圧縮できる。だが圧縮は情報のボトルネックになり、長距離依存は勾配消失・表現の上書きで薄れる。畳み込みは局所窓しか見ない。Transformer 系の答えは、各層で全位置ペアの類似度を再計算する自己注意だ——ただしその「全」も、実際には有限長の窓と、デコーダでは未来を見ない因果マスクの中での「全」である。無限記憶への扉はここにはない。

窓のどこからでも借りられなければならない

次トークン予測は各位置に分布を出させるが、トークン3の情報をトークン30へどう届けるかは、それ自体では言わない。初期の系列モデルは接頭辞全体を固定サイズの再帰状態に一歩ずつ押し込んだ——局所構文には有用だが、始めの指示や固有名が末尾の語を支配しなければならないときに脆い。畳み込みや固定幅窓はもっと悪い——カーネル外は単に届かない。失敗は構造的だ。固定局所特徴と脆い逐次状態では、早い指示を遅い語へルーティングできない。 必要なのは、各位置が接頭辞全体を読み直し、どれだけ各早いベクトルから借りるかを微分可能な重みで決める機構。

クエリ–キー類似度

問題を「何を探しているか？」と「各隣が何を提供するか？」に分ける。位置 i でモデルは現在の埋め込みからクエリベクトル q_i を形成する。文脈内の各位置 j（i 自身を含む）はキーベクトル k_j を公開する。スカラースコアは内積 q_i · k_j——二ベクトルの方向が揃えば大きく、揃わなければ小さい。直観的には、クエリがソフトな問い（「この構文役割に合うのは？」「主語はどこで導入された？」）を投げ、各キーがそのトークンが関連しうるかを広告する。これらのスコアは離散検索ではない——注意はソフトな混合であり、硬い事実ポインタではない。 単一キーが独裁的に「勝つ」わけではない；すべての位置が異なる重みの候補のまま残る。

ソフトマックス重み

生の内積は広い範囲を取りうるので、各クエリについて位置間にソフトマックス重みを適用する——スコアを指数化し、和で割り、非負で和が一のベクトルを得る。位置 j の重み α_{ij} は「更新の何割をトークン j から借りるか？」への答え。動詞位置のクエリが文主語のキーと揃えば、その主語インデックスに質量が行く。早い指示トークンが後のスロットとパターンを共有すれば、質量は窓を逆方向に流れうる。ソフトマックスの尖りはキー/クエリスケーリングを通じて間接的に学習される；尖った峰は「主にここを聞け」、平坦な峰は「多くを混ぜろ」。重みは各層・各ヘッドで再計算される——「参照先」と永久ラベルされた辺はない。

デコーダのみの生成モデルでは、位置 i のクエリは j ≤ i にだけ注意する——未来のトークンはまだ存在しない（または見てはいけない）からだ。これが自己回帰と注意の接合部だ。学習時は正解続きも同時に見えるが、マスクで未来を隠し、各位置で「左だけから次を当てよ」というゲームを保つ。推論時は生成済みトークンだけが左側に増えていく。

値の混合

キーはいくら借りるかを決め、値ベクトル v_j は何を借りるかを決める。各位置はキーと並行して値ベクトルを持つ。位置 i の更新表現は値の混合：Σ_j α_{ij} v_j、文脈窓からの値ベクトルの加重平均。この自己注意ミックスの後、位置 i のベクトルは接頭辞全体から再構成され、直近隣か圧縮メモリセルだけからではない。層を積むとレシピが繰り返される——層を積むことはミキサーを積むこと；一ヘッドは知識グラフではない。 深さは逐次再混合パスを加え、ポインタ付きの事実棚を加えるのではない。

実装では「ヘッド」が並列に複数走り、各ヘッドが異なる種類の類似度（構文、共起、長距離依存の近似など）を学習しうる。それでも各ヘッドの出力は値の混合であり、データベースのキー paper_id=12345 への硬い参照ではない。捏造引用が起きうる理由の一つがここにある——Bibliography 方言のトークン列は混合で再構成される「もっともらしい平均」として出うるが、外部の文献レコードへのポインタではない。

再構成：三位置を手計算

三トークン——"It" " sat" " quietly" の ID——と、簡略化した2次元値ベクトル：

位置トークン値ベクトル v
0It[1.0, 0.0]
1sat[0.0, 1.0]
2quietly[1.0, 1.0]

位置2（quietly）のクエリが位置0–2にソフトマックス重み [0.1, 0.2, 0.7] を与えたとする。位置2の新ベクトルは：

0.1 · [1,0] + 0.2 · [0,1] + 0.7 · [1,1]
= [0.1, 0.0] + [0.0, 0.2] + [0.7, 0.7]
= [0.8, 0.9]

出力は単一入力行の硬いコピーでもない——類似度加重平均だ。Vaswani らはこの並列混合ブロックを Transformer の核として導入した（Vaswani et al., 2017）。seq2seq におけるエンコーダ状態への注意は、より狭い形で同じソフトルーティングの発想を示していた（Bahdanau et al., 2014）。

Transformer が RNN より長距離依存で成功した理由を、層の言葉で言い直す。固定幅の再帰状態に早いトークンを押し込む代わりに、各層が窓内全位置から値を借り直す——勾配は注意重みを通じて遠い位置にも流れやすい。ただし計算量は窓長 n に対しおおむね O(n²) だ。だから「無限に読む」ではなく、有限コンテキスト窓という物理が製品設計に戻ってくる（第8章）。

位置0の指示トークン「Answer in JSON only」が、位置400の出力形式に影響しうるのは、この重み付き混合のおかげだ——硬いポインタで JSON スキーマを参照しているのではない。混合後のベクトルが、次トークン分布で { や "key": に質量を移す。

シミュレーションを開く — 自己注意は、トークン表現を文脈内の値ベクトルの類似度つき加重平均で置き換える。

出口命題

自己注意は、各位置を文脈内の値ベクトルの類似度加重平均として再構成する。

混合の後も、ネットワークは最終位置で語彙 ID 上の分布を保持している——まだ一つの発話続きにコミットしていない。訓練は尤度を最適化した；推論は一度に一 ID を出し、付け足し、再実行しなければならない。スコア付き分布は、まだ発話された返答ではない——その隙間は次章のサンプリングループの所属だ。

発話は1トークンずつサンプリングされる

再生成を押すと、形容が変わり、箇条書きの順序が入れ替わり、新しい冗談の書き出しが付く——それでも要点はしばしば残る。この日常ミステリーは、信念の間でモデルが「考えを変えた」のではない。自己注意ミックスの後も、各ステップは語彙上のスコアの雲しか持たない。ユーザーは読めるトークン列を要する；機械はその雲を不可逆な一選択に変え、それを条件にしなければならない。常に最高スコアだけ取ると退屈で反復的になる。語彙全体から一様に引くと意味不明な文字列になる。強いる中間道は、同じ条件付きスコア上の構造化されたランダム性だ。

入口で置いた再生成の言い回し差を、ここで最初に名指しする。同じプロンプト、同じ重み、同じ温度でも、各ステップの抽選結果が変われば、形容詞・例示・段落の入り方は揺れる。要点（骨格トークン）が残るのは、条件付き分布の質量が同じモード付近に集中しているからだ。「考えを変えた」のではなく、同じ分布から別のサンプル経路を辿っただけと見るべきだ。

再生成ミステリーの長い再構成

同じユーザー質問、同じモデル重み、温度 T=0.7 固定。一回目の生成でトークン列が A, B, C, …, Z、再生成で A, B', C, …, Z' になったとする。A と Z 付近は高確率モード——ほとんどのサンプルで同型。中間の B' や装飾語は中〜低確率から引かれ、経路が分岐する。ユーザーが「信念が変わった」と感じるのは、自己回帰サンプリングが各ステップで独立にサイコロを振っているのに、条件付き分布の峰が安定しているからだ。温度を下げると B' の揺れは減るが、創造性ではなく尖りの調整に過ぎない。事実の有無は logits の inventory に既に書かれている——再生成はその inventory を書き換えない。

自己回帰ループ

生成はループであり、一発段落ダンプではない。ステップ t でモデルはプロンプトと既に出力したすべてのトークンを受け取り、ミキサーのスタックを走らせ、次位置だけの logits を出す。一トークンが選ばれ、付け足され、ループが繰り返されるまで停止条件まで。これが Attention Is All You Need で述べられ、 GPT-2 全体で使われる自己回帰ループだ——各新 ID は後続すべての永久文脈になる。完成エッセイにコミットしてから打つ「計画モジュール」は別にない；局所選択が大域形状を積み上げ、早い誤りを後のトークンが糊塗しなければならないことも含む。

停止条件も層の一部だ：特殊な 停止トークンID がサンプルされたらループ終了、最大長に達したら強制終了。どちらも「答えが完成した」という理解ではなく、実装ルール。だから途中で切れた箇条書きや、余計に続く謝罪文も、分布と停止規則の産物として説明できる。

ソフトマックスサンプリング

logits はソフトマックスで次トークン分布になる——指数化、正規化、和が一の確率。ソフトマックスサンプリング（自己回帰サンプリングの核心）はその分布から ID を引く（または切り詰め変種）。argmax は常に最頻値を取る退化例。nucleus（top-p） や top-k は、累積確率または上位 k 個以外をゼロにしてから引く——尾を切るだけで、新しい事実は注入しない。製品 UI の「創造性」スライダーが触るのは、しばしばこの尾の切り方と温度の組み合わせだ。玩具語彙 {the, a, cat, sat}、logits [2.0, 0.5, 1.0, -1.0] での再構成：

probs ∝ exp(logits)
≈ [0.54, 0.12, 0.20, 0.03]  （正規化後；数値は示意）

一度引く：the かもしれない。同じベクトルから再び引く：cat かもしれない。再生成は各ステップでこのサイコロを振り直すこと——第二の世界モデルに相談することではない。再生成の言い回し差はここで回収される：骨格トークンは高確率で繰り返し、装飾的低質量トークンだけが揺れる。

温度の尖り

製品は softmax の前で同じ logits を形作る制御を露出する。logits を温度 T > 0 で割る。T が小さいと峰が尖り、最頻値が支配する。T が上がると分布が平坦になり、低確率トークンに質量が回る。それが温度の尖り——スコアの再形作であり、知識注入ではない。モデルが正しい稀な事実に一度も確率を割り当てていなければ、T を上げても信頼できる引用は生まれない——流暢な誤りがより多様になるだけ。

シミュレーションを開く — 温度を上げると次トークン分布が平坦になり、低確率トークンが選ばれやすくなる。

二 logits [3.0, 1.0] の対比：

Tp(トークンA)p(トークンB)
0.5~0.98~0.02
1.0~0.88~0.12
2.0~0.73~0.27

同じ根底スコア；異なる尖り；異なるサンプル多様性。

再生成の言い回し差のミニ再構成：接頭辞「Explain gravity simply」で、最初の生成が「Imagine a trampoline…」、再生成が「Picture a heavy ball…」と始まったとする。トランポリンとボールは、どちらも「簡単な重力説明」接頭辞のあとコーパス上ありそうな比喩トークン——どちらも中〜高確率の尾から引かれうる。一方、「重力はカーブ空間の測地線…」のような専門的続きは、温度が低ければ最初から除外され、要点の骨格（「重力＝質量が引く／時空の曲がり」等）は高確率モードとして残る。ユーザーが感じる「同じ説明、違う例え」は、この層の正常動作だ。

混同を避ける：温度は「創造性」ラベルの下で事実を足さない。 サンプルされたトークンは不可逆な文脈——後段は、新しい生成かループ外の編集なしには取り消せない。

製品の「創造的／正確」トグルを層で翻訳すると：創造的側は、しばしば温度の尖りを下げる（分布を尖らせる）のではなく、上げる（平坦化して低確率トークンを許す）か、top-p を広げる。正確側は argmax に近づける——反復は増えるが、捏造引用のリスクは「創造性」より分布的もっともらしさの話として残る。サンプリングをいじっても、存在しない DOI を真実の棚から引いてくるわけではない。

出口命題

返答は左から右へサンプリングされる；温度は同じスコアを再形作するだけで、事実を足さない。

文書続きは依然として Web 風テキストのもっともらしさを最適化する。ユーザーが欲しいのは指示追従と評価された有用性——それは次の圧であり、別のサンプリング技ではない。

プロンプトがプログラムになり、嗜好が方針を形作る

自己回帰サンプリングは任意の接頭辞——半分書いたブログ、Python ファイル、チャットログ——を続けうる。それでも文書完了だ。チャット製品を使う人はもっと鋭いことを欲する：形式の例を三つ見せてパターンを吸収させ、箇条書き計画を頼んで箇条書きを得、有害要求を拒否させて拒否スタイルを見る。それらは softmax の無料ギフトではない。同じエンジンに積まれた二つの連続圧から来る——プロンプト自体をスケールで使えるプログラムにすること、そして人間の嗜好で方針を形作り、既定界面を自動補完ではなく助手に感じさせること。

文脈内パターン完了

コンテキスト窓にタスク記述と少数の入出力例があると、十分訓練されたモデルはしばしば示された形式で続ける——勾配更新なしに。Language Models are Few-Shot Learners (GPT-3) がこのレジームを明示する：プロンプトが「プログラム」を供給し、文脈内パターン完了は依然として条件付き次トークン予測だ。再構成（模式）：

プロンプト:
  translate English to French
  sea otter => loutre de mer
  cheese => 
モデルは高い質量で続ける: fromage

重み行列はリクエスト中にその正確な三つで微調整されたわけではない。続きは、デモトークンを読んだ後の次トークン分布が割り当てるもの。混同しない：プロンプトを読まずに学習ではない。例そのものが条件づけだ。小モデルが同じ手品に失敗するのは、別のメタ学習魂がないからではなく、事前訓練中に「プロンプトを埋める形式として扱え」という統計パターンを十分吸収していないから、ということが多い。

実務での含意：few-shot 例は「教えたつもり」でも、モデル側は文脈内パターン完了として読む。例の書式（区切り記号、ラベル語、改行）が訓練分布とズレると、例を増やしても形式が崩れる。勾配更新なしに「学習した」ように見える振る舞いは、すべて条件付き次トークン予測の読み取りに還元できる。

指示微調整

生の事前訓練テキストは文書だらけで、丁寧な Q&A ではない。強力な few-shot モデルでも、裸の命令形を無視したり、スクレイプページのように脱線したりする。指示微調整は教師あり段階を加える——多くの (指示, 望ましい応答) ペアを見せ、応答側で次トークン予測を続ける。部品表は小さい——同じ架構、同じサンプリングループ——だが分布は「指示形接頭辞を見たら助手形続きを出せ」方向へ移る。依然として尤度訓練；事実チェッカーは載らない。界面——指示追従——をよりありそうにする。

ここでも損失は「正しい答え」ではなく「データセットに書かれた助手形の続き」だ。教師データに誤情報が含まれれば、指示微調整はそれも再現しうる。拒否文のテンプレート、箇条書きの癖、過度な丁寧さ——これらは助手方言の尤度最大化であり、倫理推論モジュールのインストールではない。

嗜好つき方針

人間は二つのありそうな答えのどちらが良いか——より明確、より安全、より簡潔、よりへつらわない——で食い違う。InstructGPT が次の強制段階を記録する：比較を集め、嗜好信号を当てはめ、（指示データの後）生成方針をそれに対して最適化する。結果を嗜好つき方針と呼ぶ。製品間の口調、拒否スタイル、「AI として…」枠組みの差の多くはここに住む。用語集からの混同禁止：真実への整合ではない。嗜好データは評価者の指示下で評価者が好むものを報酬する；流暢な虚偽は、役に立って見えれば依然勝ちうる。捏造引用は、評価者が毎回明示的に罰しない限り、局所もっともらしさで依然高得点になりうる——嗜好訓練は口調を変えても、コア損失が世界状態チェックを要求していない事実は変えない。

InstructGPT パイプラインを層で読むと： (1) 事前訓練 LM＝次トークン予測、(2) 指示微調整＝助手形続きの尤度、(3) 比較データからの報酬モデル＝人間が好む並び、(4) PPO 等で方針更新＝嗜好つき方針。どの段階も「この文が事実か」を直接ラベルしない限り、引用の真偽は自動では買えない。製品が「安全」に感じるのは、拒否続きや慎重な言い回しが報酬されやすいから、という説明の方が、トークナイザ内蔵の良心説より層に沿う。

日常の利害——一製品は拒否し別は完了する、「Act as…」で口調が変わる——は、トークナイザ内の道徳モジュールを発明する前に、嗜好つき方針と指示データが違うかを問うべきだ。

Brown らの GPT-3 論文は、スケールが文脈内パターンを実用域に押し上げうることを示した（Brown et al., 2020）。Ouyang らの InstructGPT は、その上に指示データと RLHF 的嗜好最適化を載せ、チャット製品の「助手感」を形作った（Ouyang et al., 2022）。どちらも、コアの次トークン予測から離れない——変わるのは接頭辞の分布と、好まれる続きの分布だ。

GPT-3 論文が示した「スケール＋文脈内例」だけでは、チャット製品の体験には足りない——ユーザーは単発の few-shot カードではなく、毎回の指示と安全基準を期待する。指示微調整と InstructGPT 系の嗜好最適化は、そのギャップを埋める製品層だ（Brown et al., 2020；Ouyang et al., 2022）。重要なのは、これらが新しい真理モジュールではなく、続きの分布を再形作するパイプラインだと見ること。アーキテクチャ図に「RLHF ブロック」と書いてあっても、中身は依然としてトークン列を条件にした次トークン予測（またはそれに等価な方針勾配）である。

出口命題

スケールがプロンプトパターンを使えるようにし、嗜好訓練が界面を形作る——どちらも真実オラクルを載せない。

流暢で役立つ答えは依然として誤りになりうるし、指示を載せる窓は有限のままだ。次章は、気分ではなく層でそれらの失敗を診断する。

流暢さが嘘をつくとき、層に名前を付ける

入口のミステリーが、今や名前を持って戻る。完璧に見えるが存在しない引用は、しばしば分布的もっともらしさの成功だ——モデルは Web の参考文献方言に似たトークン列を出すが、論文が存在するかの外側チェックはない（GPT-3 も InstructGPT も、尤度＋嗜好システムの予想される限界として不真実・偏った続きを扱う）。巨大貼付で制約の最初の箇条書きが消えるのは、しばしば有限コンテキスト窓——素朴な自己注意ミックスは長さに対し二次（Attention Is All You Need）、製品は切り詰めや圧縮を行い、早い ID は単にミキサに入らない。長い貼付で先頭指示が消えるミステリーはここで回収する。どちらも単一の幽霊的「AI の気分」として診断すれば能動性を無駄にする。層に名前を付けよ。

捏造引用の長い再構成

ユーザー：「Smith (2019) の underwater basket weaving の DOI を教えて」と入力。モデルは 著者名トークン列] + (2019) + [ジャーナル風語彙] + doi.org/10.xxxx/yyyy のような続きに高い尤度を置きうる——Web 上の参考文献段落で何度も見たパターンだから。各断片は「もっともらしい」；結合の真偽は損失に登場しない。ユーザーが DOI をクリックできないまま信じるのは、外側の検証ループを省略した結果だ。InstructGPT 系でも、評価者が「捏造引用」を体系的に低評価しない限り、同型の失敗は残りうる（[Ouyang et al., 2022）。

長い貼付の長い再構成

システムプロンプト 500 トークン、マニュアル抜粋 8000 トークン、ユーザー質問 200 トークン——合計が製品上限 8192 を超えれば、何かが切り落とされる。切り方は製品依存（先頭優先、末尾優先、要約）だが、切られたトークンは自己注意ミックスに入らない。先頭に書いた「JSON のみ返せ」は、末尾の質問と長い文書に押し出され、実効文脈から消えた——不服従ではなく不在。対策は層に沿う：制約の繰り返し、文書の要約、検索で関連部分だけを窓に載せる。

分布的もっともらしさ

次トークン予測は訓練風テキストへの局所適合を報酬する。真の答えが稀・未知・窓にないとき、最高質量の続きは文法的で自信ありげで、それでも誤りうる。それが分布的もっともらしさとしての幻覚——私的な世界モデルを持つ欺く主体の証拠ではない。再構成：「Smith & Lee の2019年水中かご編み DOI…」を頼み、DOI 形の文字列が現れるのを見よ。断片（年、著者パターン、DOI 構文）は高質量；その結合は検証されなかった。捏造引用は、この層の典型症状だ。

もう一つの典型：数値集計。モデルはスプレッドシートを持たず、トークン列だけを見る。「Q1–Q4 の合計」でも、桁の足し算は外側の検証ループ（Python、Excel）向きで、次トークン予測だけでは信頼できない。それでも報告書風の数字列は、分布的もっともらしさで出やすい——だから「Excel で再計算せよ」は贅沢ではなく層の設計に沿った要求だ。

有限コンテキスト窓

展開中の LM はすべて境界付きトークンリストに注意する。GPT-3 の公開構成は2048トークン文脈（Brown et al.）；現代製品は予算を拡大するが、予算は予算のまま。システムプロンプト、検索ドキュメント、ユーザー貼付が競合すれば、何かが落ちる。以前のターンを「覚える」チャット UI は外側システム——要約、埋め込み、データベース——の話であり、コアミキサが無限履歴を保持する証拠ではない。早い指示が無視されたら、不服従を疑う前に切り詰めと位置を検査せよ。

長文貼付の実務テクニック（層の理解として）：重要制約を末尾にも繰り返す——自己注意は双方向に見えるが、実装と製品の切り詰めはしばしば先頭が落ちやすい。要約・検索で外側から窓を組み立てるのも、有限コンテキスト窓の補償であり、モデル内蔵の無限記憶ではない。RAG（検索拡張生成）を使う場合も、結局ミキサに入るのは有限トークン列だけだ——検索品質と窓配分がボトルネックになる。

「ロングコンテキスト」と謳う製品を層で読む：窓長 n が増えても、自己注意が混ぜられるのは有限長の列という契約は同じ。128K トークンも有限だ。さらに、全ペア注意の計算・メモリコストは n が増えると実装側の制約にぶつかり、近似注意・階層的圧縮・要約キャッシュなどの外側または準外側の工夫が入る。それらを「モデルが全部覚えている」と混同しない——覚えているのではなく、今この forward に入った ID 列だけがミキサの入力だ。

シミュレーションを開く — 自己注意が混ぜられるのは有限長のトークン列だけ。窓から落ちた先頭指示は不在である。

外側の検証ループ

真偽追跡は、生成エンジンの外側に巻く任意インフラだ——開いて確かめる引用付き検索、コード実行、単体テスト、人間レビュー、計算機。これを外側の検証ループと呼ぶ。交差エントロピーから暗示されない。教育上の要点は、LM 損失がそれを買うと期待するのをやめること。タスクが検証可能なら、散文の外で確かめよ。検証不能なら、流暢さを判決ではなく草稿として扱え。

具体例：数学の最終答えは計算機か筆算で；文献は DOI や Google Scholar で；コードは実行とテストで；医療・法律の判断は資格者と規制ソースで。外側の検証ループは贅沢ではなく、次トークン予測が最初から買わなかった能力の後付け配線だ。

症状最初に疑う層弱い説明（拒否）
偽引用 / 捏造引用分布的もっともらしさ「わざと嘘をついた」
長い貼付で早いルール消失有限コンテキスト窓「不服従した」
再生成で言い回しだけ変化自己回帰サンプリング「考えを変えた」
製品ごとの拒否 / 口調シフト嗜好つき方針「トークナイザが道徳を学んだ」
正しい数値や出典が必要外側の検証ループ「温度を上げれば真実が見つかる」

予告ミステリーの回収：捏造引用 → チェックなしのもっともらしさ；再生成の言い回し差 → サンプリング（前章）；先頭指示無視 → 有限コンテキスト / 混合予算。

診断の順序にも癖をつけよ。まず「検証可能か？」——可能なら外側の検証ループ。次に「制約は窓に入っているか？」——長い貼付なら有限コンテキスト窓。次に「骨格は同じで言葉だけ違うか？」——再生成なら自己回帰サンプリング。最後に「口調・拒否だけが違うか？」——嗜好つき方針。どの段階でも、最初に「わざと嘘をついた」と決めつけない——それは層地図を捨てるショートカットだ。

梯子の上に立っている——トークンID、次トークン予測、自己注意ミックス、自己回帰サンプリング、文脈内パターン完了、嗜好つき方針、または外側の検証。記事の締めは、論文を開き直さず走れる判断規則を結晶化する。

出口命題

失敗は予測・混合・サンプリング・嗜好・欠けた外側チェックに局所化される——単一の「AI の気分」ではない。

残るのは、新しい日常ミステリーが来たとき、まず層を選び、再プロンプト・文脈短縮・サンプリング変更・製品切替・モデル外検証のどれをするか決めるポケットマップを持ち帰ることだ。

雰囲気ではなくポケットマップを持ち帰る

論文タイトルを暗唱する必要はない。能動性を保つのに必要なのは、層の地図と、幽霊と交渉する前にどの層が失敗したかを問う習慣だ。

判断規則

  1. 流暢だが検証可能なら誤り → まず分布的もっともらしさを疑え。損失は局所的典型テキストを報酬した；世界状態チェックは要求しなかった。引用を開け、コードを走らせ、クリックできる出典を求めよ。
  2. 長い貼付で早い指示が消える → 有限コンテキスト窓と、製品が切り詰めた・要約したものを疑え。制約を末尾近くに置き直し、短くし、検索せよ——温度だけ上げるな。
  3. 再生成で言い回しは散るが骨格は同じ → 同じ条件付き分布上の自己回帰サンプリングを見ている。温度の尖りは多様性を変える；知識在庫は変えない。
  4. 口調・拒否・「AI として…」枠が製品で違う → 嗜好つき方針と指示データを問え。界面の形成であり、トークナイザが道徳を学んだ証拠ではない。
  5. 真実が要る、草稿の散文ではない → 外側の検証ループを足せ。検索・ツール・テスト・人間はマナーではなく、LM 損失が買わなかった欠けたチャネルだ。

#### 判断規則を物語で試す

ケース1（捏造引用）： レポート用に「2018年 Nature の CRISPR 論文を三つ」と頼み、完璧な著者リストと DOI が返った。層：まず外側の検証ループ（DOI 解決）。ダメなら分布的もっともらしさ——損失は Nature 風の続きを報酬しただけ。再プロンプトで「存在確認済みのみ」は、嗜好つき方針が拒否を学んでいれば効きうるが、保証ではない。

ケース2（再生成）： 同じメール下書きを二回生成し、件名は同じ、本文の挨拶だけ変わった。層：自己回帰サンプリング＋温度の尖り。信念更新説は捨てる。

ケース3（長い貼付）： 80ページ PDF 全文と「第3章だけ要約せよ」——要約が第5章寄り。層：有限コンテキスト窓。先頭の章番号指示が窓から落ちたか、注意予算が後半に偏ったかを疑う。対策：章だけ抽出してから投入。

いつ梯子を降りるか

ブレスト、言い換え、形式探索なら高所に留まれ——文脈内パターン完了と嗜好つきチャットで足りる。壊れたら困ることが賭けられているとき降りよ：コードが変な分割で壊れるなら ID とトークン化；制約がずれるなら自己注意ミックスと窓長；出力が反復的かカオスならサンプリング；製品挙動が変わったなら嗜好層；主張が世界と接触して生き残る必要があるならいつでも外側検証。

間違った手は同じ層を強く押すこと——切り詰め窓に長い懇願プロンプト、欠けた事実に「もっと創造的に」。

生活への配置

声に出して、どの層で操作しているか言え：

  • 言語を トークンID と 埋め込み参照 に入れている
  • 次トークン予測を訓練／利用している
  • 自己注意ミックスで制約をルーティングしている
  • 自己回帰サンプリングでテキストを出している
  • 文脈内パターン完了で操縦している
  • 嗜好つき方針に出会っている
  • 外側の検証ループで確かめている

層に名前が付けられなければ、雰囲気領域に戻っている——流暢なシステムが能動性を取り戻すのはまさにそのときだ。

場面別の短いドリル： 論文引用を求められた → まず外側の検証ループ、次に分布的もっともらしさ。同じプロンプトで二回目が違う → 自己回帰サンプリングと温度の尖り。100ページ PDF を貼ってルール無視 → 有限コンテキスト窓と先頭指示の再配置。製品 A は拒否・製品 B は答える → 嗜好つき方針。コードが一文字違いで壊れる → トークンID 分割の確認。

週次で一つでよい：今週触った LLM 出力の失敗例を一つ選び、声に出して層名を三つまで付ける。 習慣ができると、記事のポケットマップは暗記ではなく反射になる。失敗例がなければ、意図的に再生成を三回押し、言い回し差を観察する——それだけで第6章の教訓が体に入る。

一息でたどる連鎖

離散テキストはトークンID とベクトルになる；次トークン予測が圧を供給する；自己注意ミックスが各位置を有限窓から借りさせる；サンプリングがスコアを不可逆な発話に変える；スケールがプロンプトをプログラムのように働かせる；嗜好訓練が助手界面を形作る；それらのどれも真実を載せない——だから重要なときは外側で検証する。

それがメンタルモデルだ。引用が完璧に見え、再生成が新しい意見のように感じ、早いルールが長い貼付から静かに消える——次にそうなったとき、それを使え。あなたの仕事は、モデルの「魂」を探すことではなく、今どの層で何が保証されていないかを宣言し、保証が要るところに外側の検証を差し込むことだ。

最後に、入口の三ミステリーを出口で一行ずつ回収する。捏造引用＝分布的もっともらしさ＋外側検証ループの欠如。再生成の言い回し差＝同一条件付き分布上の自己回帰サンプリング。長い貼付で先頭指示が消える＝有限コンテキスト窓による不在。これらを一つの「AI 気分」に畳まない限り、あなたはプロンプト・製品・検証のどこを触るべきか選べる——それがこのツアーの契約の完遂だ。第一原理から組み立てた以上、暗記ではなく層に降りる習慣が唯一の持ち帰りになる。次に LLM 出力に違和感を覚えたら、まず幽霊ではなく層を疑え。トークンID、次トークン予測、自己注意ミックス、自己回帰サンプリング、文脈内パターン完了、嗜好つき方針、外側の検証ループ——この七つの名前が、雰囲気を地図に変える。それが「LLM, from First Principles」の約束だ——魔法でも詐欺でもなく、制約の連鎖として読めること。能動性は、その連鎖を口に出せるかどうかから始まる。
